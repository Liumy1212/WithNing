# 打卡监督系统 数据库文档

## 1. 数据库设计

### 1.1 用户表 (`user`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 用户ID |
| `username` | `VARCHAR(50)` | `NOT NULL, UNIQUE` | 用户名 |
| `password` | `VARCHAR(100)` | `NOT NULL` | 密码（加密存储） |
| `nickname` | `VARCHAR(50)` | `NOT NULL` | 昵称 |
| `email` | `VARCHAR(100)` | `NOT NULL, UNIQUE` | 邮箱 |
| `phone` | `VARCHAR(20)` | `UNIQUE` | 手机号 |
| `role` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'normal'` | 用户角色（admin: 管理员, normal: 普通用户） |
| `points` | `INT` | `DEFAULT 0` | 用户积分 |
| `create_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `update_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

### 1.2 任务表 (`task`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 任务ID |
| `title` | `VARCHAR(100)` | `NOT NULL` | 任务标题 |
| `description` | `TEXT` | `NOT NULL` | 任务描述 |
| `type` | `VARCHAR(20)` | `NOT NULL` | 任务类型（daily: 每日任务, stage: 阶段任务） |
| `reward_points` | `INT` | `NOT NULL` | 奖励积分 |
| `deadline` | `DATETIME` | | 任务期限（阶段任务必填） |
| `publisher_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES user(id)` | 发布者ID（管理员发布） |
| `status` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'approved'` | 任务状态（approved: 已通过, pending: 待审核, rejected: 已拒绝） |
| `create_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `update_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

### 1.3 用户任务表 (`user_task`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 记录ID |
| `user_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES user(id)` | 用户ID |
| `task_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES task(id)` | 任务ID |
| `completed` | `BOOLEAN` | `DEFAULT false` | 是否完成 |
| `complete_time` | `DATETIME` | | 完成时间 |
| `create_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `update_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

### 1.4 任务申请表 (`task_submit`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 申请ID |
| `user_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES user(id)` | 申请人ID（普通用户） |
| `title` | `VARCHAR(100)` | `NOT NULL` | 任务标题 |
| `description` | `TEXT` | `NOT NULL` | 任务描述 |
| `type` | `VARCHAR(20)` | `NOT NULL` | 任务类型（daily: 每日任务, stage: 阶段任务） |
| `reward_points` | `INT` | `NOT NULL` | 奖励积分 |
| `deadline` | `DATETIME` | | 任务期限（阶段任务必填） |
| `status` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'pending'` | 审核状态（pending: 待审核, approved: 已通过, rejected: 已拒绝） |
| `reviewer_id` | `BIGINT` | `FOREIGN KEY REFERENCES user(id)` | 审核人ID（管理员） |
| `review_comment` | `VARCHAR(200)` | | 审核评论 |
| `create_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 申请时间 |
| `update_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

### 1.5 积分记录表 (`point_record`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 记录ID |
| `user_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES user(id)` | 用户ID |
| `points` | `INT` | `NOT NULL` | 积分变动值（正数为增加，负数为减少） |
| `description` | `VARCHAR(200)` | `NOT NULL` | 积分变动描述 |
| `type` | `VARCHAR(20)` | `NOT NULL` | 积分变动类型（task: 任务奖励, penalty: 惩罚, exchange: 兑换） |
| `create_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 创建时间 |

### 1.6 商品表 (`product`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 商品ID |
| `name` | `VARCHAR(100)` | `NOT NULL` | 商品名称 |
| `description` | `TEXT` | `NOT NULL` | 商品描述 |
| `price` | `INT` | `NOT NULL` | 商品价格（积分） |
| `image_url` | `VARCHAR(255)` | `NOT NULL` | 商品图片URL |
| `stock` | `INT` | `NOT NULL, DEFAULT 0` | 商品库存 |
| `status` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'active'` | 商品状态（active: 上架, inactive: 下架） |
| `publisher_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES user(id)` | 上架管理员ID |
| `create_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `update_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

### 1.7 兑换记录表 (`exchange_record`)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|----------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 记录ID |
| `user_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES user(id)` | 用户ID |
| `product_id` | `BIGINT` | `NOT NULL, FOREIGN KEY REFERENCES product(id)` | 商品ID |
| `price` | `INT` | `NOT NULL` | 兑换价格（积分） |
| `exchange_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 兑换时间 |

## 4. 数据库初始化脚本

```sql
-- 创建用户表
CREATE TABLE `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL,
  `password` VARCHAR(100) NOT NULL,
  `nickname` VARCHAR(50) NOT NULL,
  `email` VARCHAR(100) NOT NULL,
  `phone` VARCHAR(20),
  `role` VARCHAR(20) NOT NULL DEFAULT 'normal',
  `points` INT DEFAULT 0,
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`),
  UNIQUE KEY `uk_email` (`email`),
  UNIQUE KEY `uk_phone` (`phone`),
  KEY `idx_role` (`role`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 创建任务表
CREATE TABLE `task` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `title` VARCHAR(100) NOT NULL,
  `description` TEXT NOT NULL,
  `type` VARCHAR(20) NOT NULL,
  `reward_points` INT NOT NULL,
  `deadline` DATETIME,
  `publisher_id` BIGINT NOT NULL,
  `status` VARCHAR(20) NOT NULL DEFAULT 'approved',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_type` (`type`),
  KEY `idx_deadline` (`deadline`),
  KEY `idx_status` (`status`),
  KEY `idx_publisher_id` (`publisher_id`),
  CONSTRAINT `fk_task_publisher` FOREIGN KEY (`publisher_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 创建用户任务表
CREATE TABLE `user_task` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `task_id` BIGINT NOT NULL,
  `completed` BOOLEAN DEFAULT false,
  `complete_time` DATETIME,
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_task_id` (`task_id`),
  KEY `idx_completed` (`completed`),
  CONSTRAINT `fk_user_task_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_user_task_task` FOREIGN KEY (`task_id`) REFERENCES `task` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 创建任务申请表
CREATE TABLE `task_submit` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `title` VARCHAR(100) NOT NULL,
  `description` TEXT NOT NULL,
  `type` VARCHAR(20) NOT NULL,
  `reward_points` INT NOT NULL,
  `deadline` DATETIME,
  `status` VARCHAR(20) NOT NULL DEFAULT 'pending',
  `reviewer_id` BIGINT,
  `review_comment` VARCHAR(200),
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_status` (`status`),
  KEY `idx_reviewer_id` (`reviewer_id`),
  CONSTRAINT `fk_task_submit_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`),
  CONSTRAINT `fk_task_submit_reviewer` FOREIGN KEY (`reviewer_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 创建积分记录表
CREATE TABLE `point_record` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `points` INT NOT NULL,
  `description` VARCHAR(200) NOT NULL,
  `type` VARCHAR(20) NOT NULL,
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_create_time` (`create_time`),
  CONSTRAINT `fk_point_record_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 创建商品表
CREATE TABLE `product` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `name` VARCHAR(100) NOT NULL,
  `description` TEXT NOT NULL,
  `price` INT NOT NULL,
  `image_url` VARCHAR(255) NOT NULL,
  `stock` INT NOT NULL DEFAULT 0,
  `status` VARCHAR(20) NOT NULL DEFAULT 'active',
  `publisher_id` BIGINT NOT NULL,
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_price` (`price`),
  KEY `idx_stock` (`stock`),
  KEY `idx_status` (`status`),
  KEY `idx_publisher_id` (`publisher_id`),
  CONSTRAINT `fk_product_publisher` FOREIGN KEY (`publisher_id`) REFERENCES `user` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 创建兑换记录表
CREATE TABLE `exchange_record` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `product_id` BIGINT NOT NULL,
  `price` INT NOT NULL,
  `exchange_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_product_id` (`product_id`),
  KEY `idx_exchange_time` (`exchange_time`),
  CONSTRAINT `fk_exchange_record_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE,
  CONSTRAINT `fk_exchange_record_product` FOREIGN KEY (`product_id`) REFERENCES `product` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 5. 数据库操作示例

### 5.1 任务相关操作

#### 5.1.1 发布任务（管理员）
```sql
INSERT INTO `task` (`title`, `description`, `type`, `reward_points`, `deadline`, `publisher_id`, `status`)
VALUES ('每日运动', '每天运动30分钟', 'daily', 10, NULL, 1, 'approved');
```

#### 5.1.2 提交任务申请（普通用户）
```sql
INSERT INTO `task_submit` (`user_id`, `title`, `description`, `type`, `reward_points`, `deadline`, `status`)
VALUES (2, '每日阅读', '每天阅读1小时', 'daily', 15, NULL, 'pending');
```

#### 5.1.3 审核任务申请（管理员）
```sql
-- 通过申请
UPDATE `task_submit` 
SET `status` = 'approved', `reviewer_id` = 1, `review_comment` = '审核通过'
WHERE `id` = 1;

-- 拒绝申请
UPDATE `task_submit` 
SET `status` = 'rejected', `reviewer_id` = 1, `review_comment` = '奖励积分过高'
WHERE `id` = 2;
```

#### 5.1.4 完成任务
```sql
-- 更新用户任务状态
UPDATE `user_task` 
SET `completed` = true, `complete_time` = NOW()
WHERE `user_id` = 2 AND `task_id` = 1;

-- 增加用户积分
UPDATE `user` 
SET `points` = `points` + 10
WHERE `id` = 2;

-- 记录积分变动
INSERT INTO `point_record` (`user_id`, `points`, `description`, `type`)
VALUES (2, 10, '完成每日打卡任务', 'task');
```

### 5.2 商品相关操作

#### 5.2.1 上架商品（管理员）
```sql
INSERT INTO `product` (`name`, `description`, `price`, `image_url`, `stock`, `status`, `publisher_id`)
VALUES ('精美笔记本', '高质量笔记本', 50, 'https://example.com/product2.jpg', 50, 'active', 1);
```

#### 5.2.2 下架商品（管理员）
```sql
UPDATE `product` 
SET `status` = 'inactive'
WHERE `id` = 3;
```

#### 5.2.3 上架商品（管理员）
```sql
UPDATE `product` 
SET `status` = 'active'
WHERE `id` = 3;
```

#### 5.2.4 删除商品（管理员）
```sql
DELETE FROM `product` WHERE `id` = 4;
```

#### 5.2.5 兑换商品
```sql
-- 开始事务
START TRANSACTION;

-- 检查用户积分
SELECT `points` INTO @user_points FROM `user` WHERE `id` = 2;

-- 检查商品信息
SELECT `price`, `stock`, `status` 
INTO @product_price, @product_stock, @product_status 
FROM `product` WHERE `id` = 1;

-- 验证兑换条件
IF @user_points >= @product_price AND @product_stock > 0 AND @product_status = 'active' THEN
  -- 扣减用户积分
  UPDATE `user` SET `points` = `points` - @product_price WHERE `id` = 2;
  
  -- 减少商品库存
  UPDATE `product` SET `stock` = `stock` - 1 WHERE `id` = 1;
  
  -- 记录兑换
  INSERT INTO `exchange_record` (`user_id`, `product_id`, `price`)
  VALUES (2, 1, @product_price);
  
  -- 记录积分变动
  INSERT INTO `point_record` (`user_id`, `points`, `description`, `type`)
  VALUES (2, -@product_price, '兑换积分兑换券', 'exchange');
  
  COMMIT;
ELSE
  ROLLBACK;
END IF;
```

## 6. 数据库优化建议

1. **使用Redis缓存**：对于频繁访问的数据，如用户积分、任务列表等，可以使用Redis进行缓存，提高查询性能。

2. **分区表**：对于积分记录表和兑换记录表等可能会产生大量数据的表，可以考虑使用分区表，按时间或用户ID进行分区。

3. **定期清理**：对于过期的任务数据和历史记录，可以设置定期清理机制，保持数据库的健康状态。

4. **读写分离**：在高并发场景下，可以考虑使用读写分离，提高数据库的处理能力。

5. **索引优化**：根据实际查询需求，优化索引设计，避免全表扫描，提高查询效率。

6. **事务管理**：对于涉及多个表的操作（如商品兑换），使用事务确保数据一致性。

7. **连接池**：使用数据库连接池管理数据库连接，减少连接开销。

8. **SQL优化**：优化SQL查询语句，避免复杂的多表关联查询，使用适当的索引。
